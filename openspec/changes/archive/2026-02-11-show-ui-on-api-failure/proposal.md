## Why

現在、VRChat API 認証エラーやアバター一覧取得失敗時に、アプリが操作不能な状態になってしまう。`App.tsx:46` および `MainAppShell.tsx:50` でエラー時に早期リターンしてエラーメッセージのみ表示するため、ユーザーは再ログインやリトライなどの操作ができず、アプリを再起動するしかない。これはユーザビリティを大幅に損なう。

API 呼び出しは一時的なネットワークエラーや VRChat サーバーの不調など様々な理由で失敗する可能性があり、グレースフルなエラーハンドリングが必要である。

## What Changes

- **API エラー時の UI 表示**: 認証失敗やアバター一覧取得失敗時でも、通常の UI コンポーネント（`MainAppShell`、ヘッダー、フッター、操作パネル）を表示する
- **エラー通知**: Mantine の `notifications.show()` で API 失敗をユーザーに通知し、エラー内容を明示する
- **キャッシュデータでの動作**: アバター一覧の取得に失敗した場合、前回取得したキャッシュデータを表示する。最新の状態ではないかもしれないが、ユーザーが部分的にでもアプリを利用し続けられることが重要
- **リトライ機能**: ユーザーが再度ログインボタンやリフレッシュボタンで API 呼び出しを再試行できるようにする
- **エラーステート管理**: TanStack Query の `isError` ステートを活用しつつ、UI レンダリングは継続する

## Capabilities

### New Capabilities
- `graceful-api-error-handling`: API 呼び出し失敗時にもアプリ UI を正常に表示し、エラー通知とリトライ機能を提供するグレースフルなエラーハンドリング機能

## Impact

### フロントエンド
- **`App.tsx`**: 認証エラー時の早期リターンを削除し、エラー時でも `<LoginForm>` または `<MainAppShell>` を表示
- **`MainAppShell.tsx`**: アバター一覧エラー時の早期リターンを削除し、エラーステートでも UI を表示（キャッシュデータを利用）
- **`useAvatarListQuery.ts`**: エラー時に `throw` せず、通知のみ表示して TanStack Query がキャッシュデータを保持
- **`AvatarList.tsx`**: 初回エラー時（キャッシュがない場合）の空配列に対するフォールバック UI を追加
- **エラー通知**: 既存の `notifications.show()` を活用し、エラー発生を簡潔に通知。キャッシュデータ利用はメイン UI の Alert バナーで明示

### バックエンド
- 影響なし（Rust コマンドは既に `Result<T, String>` を返しており、エラーハンドリングはフロントエンドで実装）

### 依存関係
- TanStack Query のエラーステート処理パターン変更
- Mantine Notifications の継続利用
